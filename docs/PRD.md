# Private Moments PRD

## 0. 项目宗旨

Moments 是一个没有观众的生活表达空间。

它是一个个人时间线应用，体验类似发朋友圈，但所有内容默认只属于你自己。没有点赞，没有评论，没有被观看的压力。写一句话，放几张照片，记录当下，然后继续生活。

你可以像刷 feed 一样回看过去，而不是像查数据库一样翻记录。

现有工具往往走向两个极端：要么太社交，总是在被看；要么太沉重，像在写日记。Moments 试图在中间找到一种更自然的方式：像社交一样轻松表达，像日记一样完全私密，像信息流一样可以沉浸浏览。

核心理念：

- **表达，而不是记录**：不需要写完整，只需要说一句。
- **默认没有观众**：内容不是为了被别人看到。
- **时间是流动的**：不是列表，而是一段持续的生活轨迹。
- **本地优先**：数据属于你自己，可同步但不依赖云端。

## 1. 产品概述

Private Moments 是一个单用户、私密、自托管的生活记录产品。它的核心体验类似私人版朋友圈：用户可以在 iPhone 上随手发布文字和图片，内容先稳定保存在本机，即使离线也不丢；当 iPhone 通过 Tailscale 或私有 VPN 连回 Mac 后，内容自动同步到 Mac 服务端作为长期归档源。

第一版不做公开社交、不做好友关系、不做多人账号体系。产品重点是低摩擦记录、可靠离线保存、私有归档和可控备份。

## 2. 目标用户

第一目标用户是项目作者本人：

- 使用 iPhone 作为主要记录入口。
- 使用自己的 Mac 作为服务器和后台管理端。
- 希望内容不依赖第三方社交平台。
- 希望外出时也能通过 Tailscale 连回 Mac。
- 后续可能开源给其他个人自托管用户。

## 3. 产品目标

- 在 iPhone 上快速发布文字和图片。
- 支持离线创建内容，本地可靠保存。
- 网络恢复且 Mac 可达后，自动同步到 Mac。
- Mac 作为最终归档源，保存数据库和图片文件。
- 支持时间线、月份归档和文本搜索。
- 支持收藏、筛选、月份快速跳转和完整图片查看。
- 支持编辑文字、发生时间、图片新增/删除和拖拽重排。
- 支持删除，并采用软删除 + 30 天后永久清理。
- Mac 后台提供运维管理能力，包括设备、服务状态、存储、同步状态和日志。
- iOS 设置页提供只读空间诊断，显示本机缓存、同步健康状态，以及 Mac 在线时的服务端存储概览。
- 数据目录清晰，便于 Time Machine 或其他外部工具备份。

## 3.1 设计原则

主时间线必须保持简洁、低干扰。新增筛选、跳转、整理和管理能力时，优先放进 toolbar menu、滑动操作、详情页或设置页；不要为了加功能把主界面堆满控件。详细原则见 `docs/DESIGN-PRINCIPLES.md`。

内容管理能力必须服务于“回到某段生活”，而不是把 Moments 变成 archive/database。输入能力必须服务于“顺手表达”，而不是把 Moments 变成 Markdown editor 或写作工具。

## 4. 非目标项

MVP 不包含：

- 好友关系、关注、多人发布或公开社交。
- 视频、语音、Live Photo、RAW 文件处理。
- 推送通知、每日提醒、纪念日提醒。
- 复杂标签、心情系统、OCR、AI 搜图。
- 可见历史版本、复杂冲突 UI 或跨设备编辑合并工具。
- 完整回收站 UI。
- 应用内一键 zip 备份导出。
- 端到端加密或应用级数据加密。
- App Store/TestFlight 分发。
- 多用户权限系统。

## 5. 平台与使用方式

### iOS

iOS App 是主入口，使用 Swift + SwiftUI 开发。用户通过 iPhone 发布、浏览、搜索、归档查看和删除内容。

第一版通过 Xcode 直接安装到自己的 iPhone，不考虑 App Store 分发。

### Mac

Mac 是服务端和后台管理端。服务端登录后自动启动，提供 API、同步、文件存储和后台 Web UI。

Mac 通过 Tailscale 或私有 VPN 被 iPhone 访问，不直接暴露公网。

## 6. 核心用户故事

### 发布内容

作为用户，我希望在 iPhone 上发布一条包含文字和图片的日常表达，这样我可以像发朋友圈一样轻松保存生活片段，但不需要面对任何观众。

验收标准：

- 可以输入文字。
- 可以从相册多选图片。
- 可以直接拍照。
- 每条动态最多 9 张图片。
- 发布时默认使用当前时间。
- 可以手动修改日期时间。
- 发布成功后内容立即出现在本地时间线。
- 输入体验可以提供轻量辅助，例如 `- `、`• `、`1. ` 的列表延续；但保存格式保持 plain text，不做 Markdown rendering，不支持 heading、bold、quote 或 link preview。

### 离线记录

作为用户，我希望没网或 Mac 不在线时也能发布，这样记录不会被网络状态打断。

验收标准：

- 离线时可以创建文字和图片内容。
- 内容保存在 iPhone 本地数据库和文件目录中。
- App 重启后内容仍可见。
- 内容标记为待同步状态。

### 自动同步

作为用户，我希望 iPhone 连回 Mac 后自动同步，这样我不需要手动维护上传流程。

验收标准：

- App 打开后自动尝试同步。
- 切到后台或锁屏后尽量继续传完当前任务。
- 文字/帖子可以先同步。
- 图片逐张上传，失败后自动重试。
- 帖子可显示部分同步状态。
- 每条内容显示小状态：待同步、部分同步、已同步、失败。
- 设置中提供同步详情页，显示队列、失败原因和重试入口。
- 设置中提供 Storage 诊断页，显示本机缓存、待同步/失败数量，并在 Mac 在线时显示服务端存储占用。

### 时间线与归档

作为用户，我希望按时间回看记录，这样可以像刷 feed 一样回到某段生活，而不是像查数据库一样翻记录。

验收标准：

- 首页支持时间线浏览，按发生时间倒序。
- 支持月份归档。
- 时间标签保持英文 UI 体系，优先使用 `Just now`、`2 min ago`、`Yesterday 2:40 PM` 等人类表达。
- 月份信息作为滚动时短暂出现的浮层提示，不作为常驻的大块结构标题。
- 支持搜索、筛选、收藏筛选、图片筛选、待同步筛选。
- 支持通过月份菜单快速跳转。
- 后续日期跳转采用月优先、日可选的轻量入口，只显示已有内容的月份和日期，不做完整 Calendar tab 或 Archive 管理中心。
- 日期跳转文案偏生活感，例如 `April 2026`、`Apr 29 · Wed`、`Today`、`Yesterday`；不显示每天数量统计，不使用数据库感的纯日期格式作为主文案。
- 新功能入口必须保持克制，不破坏主时间线的简洁感。

### 搜索

作为用户，我希望能搜索文字内容，这样我可以快速找到过去的记录。

验收标准：

- iPhone 本地搜索已同步的文本元数据。
- 离线时可搜索本地已有文本。
- Mac 后台也提供服务端搜索。
- MVP 不做图片 OCR 或 AI 搜图。

### 查看和编辑动态

作为用户，我希望能点进一条动态查看详情，并在发错内容时直接修改，这样私人记录可以长期保持准确。

验收标准：

- 时间线点动态进入详情页。
- 详情页支持查看完整文字、发生时间、图片和同步状态。
- 详情页支持完整图片浏览。
- 详情页支持删除整条动态，并需要二次确认。
- 时间线左滑删除保留，但也需要二次确认。
- 详情页右上角进入编辑页。
- 编辑页支持修改文字、发生时间、新增图片、删除图片、长按拖拽重排图片。
- 保存时以编辑页最终图片列表作为新状态。
- 可以只有文字，也可以只有图片，但不允许文字和图片都为空。
- 编辑直接覆盖原动态，不做可见历史版本。
- 编辑本地先保存，随后通过同步队列上传到 Mac。
- 同步中或部分同步的动态暂不允许编辑；已同步或失败的动态允许编辑。
- 单张图片删除采用软删除，并由服务端 30 天后清理文件。
- 新增图片上传失败时，动态显示为部分同步。
- 编辑草稿按动态保存；再次打开时询问继续编辑草稿或丢弃草稿。
- 保存成功或用户主动丢弃时清除编辑草稿。
- 详情页弱显示 iPhone 本地最后编辑时间。

### 删除

作为用户，我希望可以删除不想保留的内容，这样记录可控。

验收标准：

- MVP 支持删除内容。
- 删除采用软删除。
- 删除操作可离线记录，并在联网后同步到 Mac。
- Mac 端保留软删除记录 30 天。
- 30 天后服务端清理数据库记录和相关图片文件。
- MVP 不要求实现完整回收站 UI。

### 设备认证

作为用户，我希望只有授权设备能访问我的服务，这样即使在 Tailscale 内网中也能防误访问。

验收标准：

- 单用户密码登录。
- iOS 登录后获得长期 device token。
- API 使用 `Authorization: Bearer <device-token>`。
- 设备 token 长期有效，可手动撤销。
- 高风险操作需要重新输入密码。
- Mac 后台可查看设备列表和撤销设备。

### Mac 后台管理

作为用户，我希望 Mac 后台显示服务状态和同步状态，这样我可以确认系统是否正常工作。

验收标准：

- 显示服务状态。
- 显示存储占用。
- 显示设备列表和撤销入口。
- 显示同步状态。
- 显示文件日志。
- 提供后台服务运维入口。
- MVP 不作为主要发布入口。

### Mac Posts 运维管理

作为用户，我希望在 Mac 后台查看和清理动态数据，这样我可以区分真机/模拟器数据、检查图片是否同步成功，并清理测试数据。

验收标准：

- 后台采用 `Overview / Posts` 顶部 tab。
- Posts 页提供列表 + 右侧详情抽屉。
- 列表显示文字摘要、发生时间、图片数、创建设备、更新设备、删除状态和基础同步状态。
- 详情抽屉显示完整文字、发生时间、创建/更新时间、创建设备、更新设备、serverVersion、媒体状态、媒体大小和 checksum。
- 详情抽屉显示图片缩略图网格。
- 点击缩略图可以全屏查看大图。
- 默认只显示未删除动态，可切换查看软删除动态。
- 支持按创建设备筛选。
- 支持按删除状态筛选。
- 支持文本搜索。
- 普通列表默认 50 条，支持 `Load more`。
- 搜索结果最多返回 100 条，第一版不做搜索分页。
- 单条删除入口只放在详情抽屉内，必须二次确认。
- 单条删除采用软删除，并通过同步事件让 iPhone 下次同步后隐藏该动态。
- 设备列表提供 `Clean posts` 危险操作，用于永久清理该设备创建的测试动态。
- `Clean posts` 必须显示候选数量和设备名，并要求输入设备名确认。
- `Clean posts` 只清理该设备创建的动态，不清理该设备仅更新过的动态。
- `Clean posts` 不自动撤销设备。
- 永久清理后，动态从 Posts 管理里彻底消失；服务端只保留让 iPhone 同步隐藏所需的最小删除事件和日志。
- 第一版不支持在 Mac 后台编辑动态正文、发生时间或图片顺序。
- 第一版不支持恢复软删除动态。

## 7. 图片策略

- 支持文字 + 图片，不支持视频和语音。
- 发布页支持相册多选 + 直接拍照。
- 每条动态最多 9 张图片。
- 上传后生成压缩展示图。
- 用户可选择是否保留原图。
- 压缩展示图默认移除 EXIF/GPS 等隐私元数据。
- 选择保留原图时，原图保留完整元数据。
- iPhone 本地保存压缩展示图 + 原图待上传副本。
- 原图上传成功后，iPhone 本地原图副本可按空间策略清理。
- Mac 长期保存压缩图和用户选择保留的原图。

## 8. 本地缓存策略

- iPhone 缓存全部文本元数据。
- iPhone 缓存已下载的展示图/缩略图。
- 旧图片可按需从 Mac 下载。
- 时间线优先从本地渲染。
- App 打开后在后台进行增量同步。
- Settings > Storage 只做诊断展示，不在 MVP 中提供清理、删除或重建缓存操作。

## 9. 草稿策略

- iPhone 支持本地草稿自动保存。
- 草稿不上传 Mac。
- 用户发布后，内容才进入同步队列。

## 10. 备份策略

MVP 采用单一数据目录，方便 Time Machine、外部硬盘、Syncthing 或其他工具备份。

默认目录：

```text
~/Library/Application Support/PrivateMoments
```

后续第二阶段补应用内一键导出 zip 备份。iCloud 的推荐使用方式是导出稳定备份包到 iCloud Drive，而不是让 SQLite 数据库直接运行在 iCloud Drive 中。

## 11. 安全策略

MVP 不做应用级加密，不做端到端加密。

第一版依赖：

- Tailscale 或私有 VPN 网络边界。
- 单用户密码登录。
- 长期 device token。
- 设备撤销。
- Mac 文件权限。

后续可考虑：

- 加密 zip 备份。
- 高风险操作重新验证。
- 更细粒度审计日志。

## 12. 开源定位

项目后续定位为个人自托管产品，而不是通用开发脚手架。

文档优先解释：

- 如何在 Mac 上安装和启动服务。
- 如何配置 launchd。
- 如何配置 Tailscale。
- 如何用 Xcode 安装 iOS App。
- 如何备份和迁移数据目录。

## 13. MVP 范围

MVP 必须包含：

- iOS 原生 App 基础发布流程。
- 文字 + 图片。
- 相册多选 + 直接拍照。
- 本地 SQLite 存储。
- 本地自动草稿。
- 离线发布队列。
- App 重启后本地内容仍可见。
- Mac 服务端 API。
- Tailscale 网络下同步。
- 文字/帖子先同步，图片逐张同步。
- 删除和软删除同步。
- Mac SQLite + 本地文件存储。
- Mac 后台管理页基础状态。
- 设备 token 鉴权。
- 数据目录和 manifest。
- 文件日志和后台日志。
- 详情页查看与编辑。
- 收藏、筛选和月份快速跳转。
- Mac Admin Posts 运维管理。

MVP 暂缓：

- 回收站 UI。
- 一键备份导出。
- iOS 多设备冲突 UI。
- 通知。
- 标签/心情。
- App Store 分发。

## 14. 核心验收标准

MVP 的端到端成功标准：

1. iPhone 在离线状态下创建一条包含文字和图片的动态。
2. App 重启后，这条动态仍然可以在本地时间线看到。
3. iPhone 通过 Tailscale 重新连上 Mac 后，App 自动同步。
4. Mac 成功保存帖子、图片元数据和图片文件。
5. Mac 后台能看到同步成功、存储占用和设备状态。
6. iPhone 上这条内容从待同步或部分同步状态变为已同步。
