# Project State

Last updated: 2026-05-08

## What This Project Is

Moments is a private expression space with no public audience. It is a personal timeline app that feels like posting to a feed, but everything belongs only to the user: no public likes, no public multi-user comments, no pressure of being watched. Feed comments are allowed only as single-user private follow-up expression.

The product should preserve four north-star ideas:

- 表达，而不是记录 — writing can be partial, quick, and lightweight.
- 默认没有观众 — content is not created for other people to see.
- 时间是流动的 — browsing should feel like returning to a living timeline, not querying a database.
- 本地优先 — data belongs to the user and can sync without depending on a cloud platform.

The iOS app, named `Moments`, is the primary capture and browsing surface. A Mac runs the self-hosted server, SQLite archive, media storage, sync API, and Admin UI.

The intended network boundary is Tailscale or another private VPN. Private owner builds may inject an additional HTTPS fallback endpoint from ignored local config when iOS is connected to another VPN and cannot keep Tailscale active, but reusable project config and public-facing docs should stay Tailscale/private-VPN-first. The project is not designed as a public cloud service.

## Current Architecture

- iOS app: SwiftUI, local SQLite, local media cache, drafts, outbox sync, and real-device install support.
- Mac server: Node.js, TypeScript, Fastify, Prisma, SQLite, local file storage, auth, sync, media, admin APIs, and static Admin UI hosting.
- Admin UI: React + Vite, built separately and served by Fastify. It is a low-frequency Mac-local operations surface, not the long-term daily settings/monitoring center.
- Shared contracts: OpenAPI route contract and sync protocol notes under `shared/`.
- XcodeGen specs: `ios/project.yml` is the canonical iOS spec for normal iOS work, and root `project.yml` mirrors it for automation that runs XcodeGen from the repository root.

## Current Product Surface

Implemented capabilities include:

- Local-first text, image, audio, video, and text + media moments.
- Photo library, camera import, and real image paste into the New Moment composer media grid.
- System Share Sheet capture through the `Save to Moments` iOS Share Extension. Shared images, videos, audio files, URLs, and text are staged in an App Group import queue and then handed to the existing Composer pipeline.
- Manual occurred date/time.
- Composer and edit drafts.
- Lightweight Markdown body editing in New Moment and Edit Moment: stored text remains Markdown source `String`, while H1/H2 render in a Typora/Obsidian-like editor with a compact H1/H2-only keyboard accessory toolbar and no Done/list buttons. Timeline, Detail, and Day Review render H1/H2 consistently.
- Plain-text list continuation in New Moment and Edit Moment for `- `, `• `, and numbered prefixes, including numbered auto-increment and empty-item exit.
- Offline outbox sync with delayed retry.
- Strict local-only mode through an iOS `Automatic Sync` setting: local create/edit/comment/tag/media work stays durable while automatic Mac server sync/upload/download/AI follow-up is suppressed until explicit `Sync Now` or re-enable. Settings reflects this mode immediately when the toggle is turned off, even if an earlier sync pass is still winding down.
- Media upload, compression, thumbnail recovery, and local cache recovery. Server media uploads write to hidden temp files first, then atomically rename only after full receipt, with stage logs for upload diagnosis.
- Timeline browsing with App Language-aware human-friendly dates and temporary floating month hint.
- Single-user private feed comments with a quiet comment action, bottom input bar, latest-two preview, in-place expand/collapse, long-press delete confirmation, and comment-aware search.
- Server-side AI media summaries for audio/video uploads, generated by the Mac server from serialized local Mac transcription plus an external summary API and shown only when ready.
- Optional audio AI title auto-insert: for new audio moments, the first ready AI summary can write only its title back as a `##` heading in the moment body.
- Local Timeline search across post text, comments, AI summary generated metadata, and historical transcript metadata; composable filters, favorites, Calendar-driven review with dynamic density/month stats/day filters, detail view, pending-safe editing, Markdown-source body copy, tappable URL cards, image gallery, and soft delete.
- Check-ins as a third iOS bottom-tab surface beside Timeline and Calendar. Check-ins store independent local-first item/entry records for recurring life activities, support one-tap empty semantic entries plus a separate note/time visibility path, and can publish individual entries into Timeline without turning them into ordinary posts.
- Settings pages for connection, sync, advanced sync, secondary Appearance/Language preferences, and tiered storage diagnostics.
- Local iOS App Language preference for `System`, `English`, and `简体中文`, plus independent AI Language preference for generated summaries/titles, both grouped under Settings > Language.
- AI Periodic Reviews foundation with Weekly Review as the first review kind: manual rolling-seven-day generation, default-off Sunday-evening auto-generation, Calendar Reviews UI, feedback, and optional publish-as-moment.
- Privacy-safe AI token usage metering for media summaries, weekly reviews, and tag fallback calls, with Settings diagnostics for Today, current week, current month, all-time totals, failures, cached input tokens, estimated requests, and feature breakdowns.
- UAT gate tracking through `docs/UAT-GATES.md`, with normal gate reporting via `npm run verify:uat-gates` and strict release blocking via `npm run verify:release-gates`. The 10 current v0.1 UAT gates were user-accepted and closed on 2026-05-07.
- Mac Admin top-level navigation is narrowed to `Archive / Overview`: Archive/recovery, promote/export/import artifacts, runtime truth, maintenance jobs, server logs, and device emergency. Posts management is no longer a visible Admin entry point; underlying posts API/legacy React code remains only as short-term troubleshooting/cleanup fallback while migration continues.
- iOS Settings > Storage & Diagnostics now carries the daily read-only Mac operations summary through two semantic modules: `Storage` contains this-iPhone storage/cache and Mac storage totals, while `Diagnostics` contains sync health, Mac server runtime, Mac operations, AI diagnostics, token usage, and tag diagnostics. Backup/restore/promote/export/import execution stays Mac Admin only.
- Device binding via stable device keys to avoid duplicate physical-device registrations.

## Current Planning Direction

M003 implements single-user `Comments` in the iOS main timeline. This supersedes the previous M002 detail-only private comments direction. Comments feel like a WeChat Moments-style feed interaction: a quiet comment icon under each moment, a bottom input bar, latest-two previews, in-place `View all N comments` / `Show less`, and long-press delete confirmation.

Comments remain private and single-user. They do not introduce a real audience, visible author identity, replies, likes, mentions, rich text, comment media, Admin UI management, or per-comment sync badges. They are local-first user data that sync through independent comment entities.

M004 covers Audio and Video Moments after the comments milestone. Audio/video are first-class moment media like images: they may stand alone or accompany text, but comments remain text-only. First version allows exactly one media kind per moment: up to 9 images, or 1 video, or 1 audio recording. The old best-effort iOS on-device speech transcription path is superseded for new clients; historical transcript schema remains only for compatibility.

M005 is the first AI layer: server-side audio/video summaries. The AI feature behaves like a quiet organizing tool, not a social actor. iOS shows only `Summary ready` after a generated summary exists; the Mac server locally transcribes the uploaded media file with `mlx-whisper`, then calls the external summary API; generated summaries are stored as removable/regeneratable metadata and viewed in a bottom sheet. New summaries use the `media-summary-v4` document model so the sheet can show a title, one-liner, headings, collapsible details, lists, and labeled `AI suggested` callouts; v4 inherits the short-title requirement from v3, with server fallback from `oneLiner`, and adds active topic tag/alias vocabulary reuse for AI topic suggestions. If no ready summary exists, no Summary entry or transcript fallback is shown. A narrow follow-up allows only new audio summary titles to sync back into `post.text` as `## <title>` when the user has not already written a leading title; summary bodies remain generated metadata. The remote-only summary pull gap has been implemented and verified on the paired iPhone, and the fresh summary/title UAT gate was user-accepted on 2026-05-07.

The current entry-point improvement adds an import pipeline rather than another publishing path: the Share Extension writes `PendingShareImport` metadata and copied files to App Group storage, then the main app opens `ComposerView` through `moments://import/<id>` and reuses existing media preparation, drafts, SQLite/outbox, and sync. Future Shortcuts, widgets, or Action Button flows should reuse this import/composer boundary instead of writing posts directly.

M006 Smart Tags implementation has landed and passed automated Mac/server/iOS verification plus real iPhone build/install/launch. Schema version 9 adds synced tag vocabulary, aliases, post tag assignments, post-level AI/user tag state, and default primary tag seeding. The iOS app supports optional composer primary tags, optional timeline primary chips, detail/edit tag views, tag/alias search, primary/topic filters, and Settings > Tags vocabulary management with batch Topic/Archived cleanup plus batch primary color edits. First-version AI automatic tags apply only to new audio moments when their first ready AI summary arrives; short audio prefers one sparse topic tag unless the content clearly has multiple high-confidence themes, and AI topic creation must reuse active topic tags/aliases before creating new vocabulary. iOS post-tag sync now applies assignment moves by assignment ID first, so server-side `merge_tag` can move an existing assignment from a source topic to a target topic without tripping `local_post_tags` unique constraints. The Smart Tags UAT gate was user-accepted on 2026-05-07.

M007 adds the iOS language foundation. Settings now routes Language into a secondary page with a local App Language preference (`System`, `English`, `简体中文`) that applies immediately to iOS-visible UI, date labels, filters, tags, comments, summary/detail/edit surfaces, and Settings. Existing private installs keep English by default; new installs default to System. Default primary tags keep one synced identity but display localized names and match both Chinese and English default names in local search/filter. AI Language is a separate local preference (`Auto`, `Chinese`, `English`) on the same secondary page and is passed to Mac-side summary generation without syncing the preference to the server.

M008 adds Calendar Review as a second bottom-tab browsing mode beside Timeline. Calendar is a local derived review/navigation surface: month view first, dynamic heatmap density scaled against the visible month's busiest day, exact count labels, small media hints, Calendar-owned month filters, a lightweight interactive `Month Stats` sheet in the navigation bar toolbar, and day tap into a full Day Review. Day Review uses 24-hour in-day timeline rows, light period separators, multi-select OR chips for media/favorites/comments, markdown-like heading rendering, uniform image thumbnails, inline audio playback, and a secondary `Timeline` action for the clearable Timeline day filter. Summary is not a standalone Calendar filter/stat dimension because Audio is the user-facing category. It supersedes the old no-Calendar-tab part of M001/D006 and removes the Timeline toolbar `Jump to date` calendar icon. It does not add compose flows, schema, sync, remote statistics, year view, AI period summaries, holidays/lunar dates, or README screenshots in v1.

M009 shifts v0.1 close-out toward owner long-term self-use after market research. Public distribution, App Store/TestFlight, and iOS standalone are not part of this milestone. Phase A implementation has landed for Admin-managed restic backup/restore, daily scheduled backups, staged restore/promote preparation with maintenance mode, pre-promote snapshot, durable maintenance jobs, and Sync Health in both Mac Admin and iOS Settings. Current v0.1 promote writes `archive/pending-promote.json` restart instructions rather than hot-swapping the live SQLite database while Prisma is connected. Phase B implementation has landed for migration-first export/import where JSON manifest/metadata is authoritative, Markdown is only a preview, and import targets a new staged archive while preserving content identity and excluding auth/session/device runtime state. Verification includes fresh SQLite setup, server/admin/iOS builds, isolated restic backup/restore/promote smoke, isolated export/import smoke, and real-device install/launch.

The owner rarely uses Mac Admin for daily operation. Future settings, monitoring, diagnostics, and safe repair controls should therefore default to iOS Settings / Diagnostics. The first migration checkpoint has moved read-only Mac operations state into iOS and removed `Posts` from the visible Mac Admin top level. Mac Admin remains for Mac-only operations such as Archive repository paths, staged promote/restart instructions, export/import artifacts, server logs, LaunchAgent/process state, filesystem permissions, and local recovery.

M010 adds AI Periodic Reviews. The first artifact is Weekly Review, but server data uses generic review kind/range concepts so monthly/custom reviews can reuse the same foundation. Weekly Review is retrospective-first with moderate reflection, uses text moments, comments, ready audio/video AI summaries, tags/favorites/time/media metadata, and leaves images as metadata-only in v1. It avoids per-claim evidence links; only `Worth Revisiting` exposes low-weight moment anchors inside the review context. Auto-generation is default off and runs Sunday evening on the Mac server when enabled. A follow-up adds `ai_usage_events` so growing AI surfaces have token/cost diagnostics without storing private AI input or output bodies.

M011 implements Pinned Moments as a quiet shortcut shelf above Timeline. Pinned moments default to a collapsed header, while pinned items remain in the ordinary chronological Timeline rows with a lightweight pin marker so pinning never makes a moment disappear from lived-time browsing. Their original `occurredAt` still drives Calendar, Day Review, search/filter results, review input, and detail identity. Pin state is lightweight post metadata (`isPinned` / `pinnedAt`) with a dedicated `update_post_pin` sync operation and `post_pin_updated` server change, not a duplicate post, tag, review, or archive-management surface. Implementation was validated in the dedicated worktree, merged to `main`, installed on the paired iPhone after a recovery checkpoint, and accepted by the user in real-use UAT on 2026-05-08.

Post-M011 feature triage keeps the roadmap conservative. Sync Doctor is the next diagnostics direction worth introducing: it should turn Sync Health evidence into a guided diagnosis and safe next action. App Lock / Privacy Mode and human-readable archive export are not needed. Shortcuts/Action Button, Monthly/Custom Review, semantic search, Web Clip, and Mac Server Watchdog are recorded as not considered for now. Stronger Markdown editing is a candidate direction, but it requires a fresh product/design boundary before code changes.

M012 adds Check-ins as a private life-activity module, not a habit tracker. The app still launches into Timeline, but the bottom tab order becomes `Timeline / Calendar / Check-ins`. Check-in items define name, icon/color, once-per-day or multiple-per-day recording, active weekdays, optional secondary tag, and default Timeline visibility. Entries store occurred time, optional note, per-entry `showInTimeline`, and soft-delete state. Timeline shows only published check-in entries as compact rows with no favorite, pin, or comments; Calendar heatmap, Day Review, Month Stats, and Check-ins History count all non-deleted entries regardless of Timeline visibility. v1 deliberately excludes reminders, streaks, missed counts, preset templates, AI tagging/OCR/transcription/summary, Mac Admin management, and separate export UI.

## Design Constraints

- The main iOS timeline must stay simple. Low-frequency controls belong in toolbar menus, swipe actions, detail views, or settings.
- Main timeline comments are an allowed M003 exception to the previous "no comments" shorthand, but only as single-user private feed interaction with controlled density.
- Audio/video support is a moment media expansion, not a comment/media-thread expansion; do not mix audio/video into comments in the first version.
- AI summaries are generated metadata, not comments or timeline prose; first-version AI may run automatically for newly uploaded audio/video media but should not auto-analyze the whole archive in the background. The only post-text mutation exception is the optional new-audio `##` title insert; full summary bodies remain generated metadata. New summary source data is a structured block model, not arbitrary Markdown.
- Content management features must help the user return to lived time, not turn Moments into an archive/database manager.
- Daily operational visibility should move toward iOS Settings / Diagnostics. Admin-only surfaces require a Mac-local reason or a documented migration note.
- Tags should support retrieval and lightweight organization without turning the timeline into a tag wall. Primary tags are expression-type metadata; topic tags are concrete searchable concepts. Timeline display is optional and limited to primary tags.
- Check-ins should stay faster than tags/composer for recurring activities. The default daily path is one tap; note/time/Timeline visibility editing is a secondary path, and item management stays inside the Check-ins surface.
- Timeline date navigation should avoid duplicate entry points. M008 Calendar becomes the primary date review surface; Timeline should not keep a parallel `Jump to date` toolbar calendar icon after Calendar lands.
- Input assistance must reduce friction for lightweight expression, not turn Moments into a full Markdown editor or writing tool. Saved post text remains Markdown source `String`; Composer/Edit currently render only line-leading H1 and H2 in a lightweight editor. Timeline/Detail/Day Review render the same limited subset. New Moment may treat pasted real clipboard images as media attachments, but Markdown image syntax remains plain text and is not parsed into media. If stronger Markdown editing is pursued, it must first define a small supported syntax set, preserve IME and long-text stability, keep comments plain, and avoid introducing a rich-text database model or full writing-tool surface by accident.
- External capture entry points must stay thin. Extensions, shortcuts, and widgets should stage inputs and let the main app composer own editing, media processing, local persistence, and sync.
- Formatting controls are H1/H2 only. List continuation edits string prefixes only. There is no rich-text storage model, inline image body model, Markdown bullet rendering, bold/quote/link preview support, or full Markdown preview surface.
- App-facing UI copy is controlled by the local App Language preference. New user-visible strings should be routed through the iOS localization layer when they appear in the main app.
- Sync cursor advancement is data-sensitive: the client must only advance after all returned server changes are applied.
- Media recovery should prefer robust batch thumbnail recovery for iOS/Tailscale reliability.
- Media upload must not commit partial files. Interrupted uploads should leave the iPhone local queue retryable, remove only server temp files, and emit enough structured logs to distinguish client/Tailscale disconnects from server write failures. iOS uploads audio/video through temporary multipart files, prioritizes `pending` media before `failed` retries, and exposes a safe `Retry Uploads` action in Storage & Diagnostics.
- Real-device Server URL should prefer Tailscale Serve HTTPS. Debug iOS builds may allow plain HTTP fallback for private development, but should not depend on `NSAllowsLocalNetworking` to cover Tailscale `100.x` addresses. If using a private fallback endpoint, expose only the iOS sync API surface, not the full Admin UI, and keep owner-specific hostnames/tunnel IDs out of tracked files.
- Personal Tailscale values and secrets must not be hard-coded into reusable code or docs.

## Current Workflow Policy

Use lightweight continuous maintenance by default. Upgrade work to milestone/slice planning when it touches high-risk areas: sync semantics, schema migrations, media storage or recovery, backup or restore, auth/security boundaries, or cross-device behavior.

Keep `main` as the fixed integration/version line. Parallel feature work should happen in dedicated Git worktrees, with one Codex thread bound to one worktree. Feature worktrees may run development, testing, builds, packaging, and UAT, but server runtime data should be isolated by default and real iPhone installs must not downgrade schema, clear app containers, lose unsynced local data, or otherwise risk existing personal data without an explicit recovery checkpoint.

Every non-trivial change must close with a short summary, fresh verification evidence, known issues or next steps, and updates to affected `.gsd` or `docs/` files.

True-device and human acceptance gaps must stay in `docs/UAT-GATES.md`. Normal work should run `npm run verify:uat-gates` to keep the list visible; release candidate readiness must run `npm run verify:release-gates`, which fails while any gate remains open. As of 2026-05-07, all current v0.1 UAT gates are closed by user acceptance.
